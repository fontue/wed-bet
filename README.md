# La Scommessa d’Amore

Шуточная свадебная prediction-market PWA на Next.js, TypeScript и Tailwind CSS. Все расчёты ведутся только во внутренних свадебных лирах; покупки, пополнения и вывода денег нет.

## Запуск

```bash
npm install
npm run dev
```

Откройте `http://localhost:3000`. Для локальной демонстрации на странице входа доступны одноразовые seed-токены:

- гость София — `guest-demo`;
- гость Миша — `misha-demo`;
- администратор Марко — `admin-demo`.

Demo-токены и кнопки быстрого входа существуют только при локальной разработке. Настоящий QR-токен обменивается на HttpOnly-сессию и сразу становится недействительным. В production специальные demo-токены отсутствуют даже в mock-хранилище.

Для входа с телефона запускайте сервер на сетевом интерфейсе (`npm run dev -- --hostname 0.0.0.0`) и открывайте адрес компьютера в локальной сети, например `http://192.168.1.20:3000`. Если админка открыта через `localhost`, задайте `APP_URL=http://192.168.1.20:3000`, чтобы QR-коды содержали доступный телефону адрес. На публичном сервере `APP_URL` должен быть HTTPS-адресом приложения.

## Экономика стаканов

Вероятность исхода определяется его долей в общем пуле. Перед подтверждением показывается коэффициент, который будет у конкретной ставки сразу после её добавления:

```text
коэффициент = (общий пул + ставка) / (стакан исхода + ставка)
```

Коэффициент продолжает меняться до закрытия рынка. После выбора победившего исхода общий пул распределяется пропорционально позициям победителей. Комиссии нет.

Стартовые вероятности обеспечиваются системной ликвидностью. Она является реальной позицией системного кошелька и участвует в финальном распределении.

## Дуэли

В разделе `/duels` гость выбирает активного соперника и одну из трёх серверных игр: старшую карту, кости или свадебный слот. Каждый вносит 200 лир, победитель получает весь банк 400 лир без комиссии.

Ожидающий вызов действует 10 минут. Отказ, отмена или истечение возвращают взнос отдельной транзакцией. Создание и принятие требуют сети и защищены idempotency key.

## Структура

- `domain/` — модели, вероятности, котировки и распределение выплат;
- `infrastructure/mock/` — заменяемый mock-репозиторий;
- `data/seed.ts` — демонстрационные пользователи, рынки и бонусы;
- `app/api/` — идемпотентные команды ставок, бонусов и админки;
- `components/` — пользовательский интерфейс без бизнес-формул;
- `public/sw.js` — кэширование оболочки и ранее открытых страниц;
- `lib/outbox.ts` — IndexedDB-очередь ставок при короткой потере сети.

Mock-хранилище живёт в памяти процесса и предназначено для первого этапа. Интерфейс доменного слоя подготовлен к замене на транзакционный PostgreSQL/Supabase-адаптер.

## Проверки

```bash
npm run typecheck
npm run lint
npm run build
npm run seed
npm run test:duels
```

Production-сборка не создаёт пользователей автоматически. Реальные аккаунты должны выпускаться только через защищённую админку.
